# 会话总结：跨境广告 AI 监控项目

**日期：** 2026-06-29

---

## 一、会话背景

用户团队有 3 名投手，分别负责不同品类，在 Facebook、Google、TikTok 三个平台投放广告。现状是全程手动登录各平台后台盯数据，容易漏看、来不及处理。核心痛点：

- 高消耗低 ROI 广告组未及时关停，持续亏损
- 高 ROI 优质广告组未及时加预算，错失放量窗口
- 低点击率素材长期投放，浪费预算，未及时通知设计师更换

---

## 二、方案设计阶段

### 2.1 整体路线确认

与用户多轮确认后，确定两阶段路线：

| 阶段 | 定位 | 状态 |
|------|------|------|
| 第一阶段 | 纯规则预警，解决漏看问题，零 AI 决策、零误操作风险 | **已落地** |
| 第二阶段 | AI 轻决策，积累数据→验证准确率→小范围自动决策 | 待后续迭代 |

### 2.2 关键业务决策

会话中逐一确认了以下关键决策：

| 决策点 | 结论 |
|--------|------|
| 企微通知方式 | 单一群 Webhook，通过 `mentioned_list` @指定投手/设计师 |
| 配置方式 | Web 界面（内网部署），投手自助操作，无需编辑配置文件 |
| 数据频率 | 调度器每小时自动刷新全部品类 + 品类页手动刷新（5分钟冷却） |
| ROI 口径 | 三平台统一使用各自后台的 ROAS 数值，无需换算 |
| 数据判断粒度 | Ad Set（广告组）层级判断 ROAS/消耗/预算；Ad（素材）层级判断 CTR |
| Campaign 绑定方式 | 投手在 Web 界面手动输入 Campaign ID 绑定到品类，系统遍历其下 Ad Set 和 Ad |

### 2.3 预警规则（4 条）

| 规则 | 触发条件 | 通知对象 |
|------|----------|----------|
| 高消耗低 ROAS | 当日消耗 ≥ 最低消耗触发值 且 ROAS ≤ 红线 | @投手 |
| 高 ROAS 可放量 | ROAS ≥ 增量线 且 消耗未达日预算上限 | @投手 |
| 预算即将耗尽 | 剩余预算 ≤ 日预算的 10% | @投手 |
| 低点击率素材 | 近 N 天 CTR ≤ 红线 且 展示量 ≥ 最低展示量 | @投手 + @设计师 |

### 4.3 数据库表（SQLite）

| 表名 | 用途 |
|------|------|
| `categories` | 品类配置：名称、投手userid、设计师userid、各项阈值 |
| `campaign_bindings` | Campaign 绑定：campaign_id、platform、category_id |
| `global_config` | 全局配置：企微 Webhook 地址 |
| `adset_snapshots` | Ad Set 层级快照（spend、roas、budget） |
| `ad_snapshots` | Ad 层级快照（ctr、impressions） |
| `alert_records` | 告警去重记录（24小时内同规则不重复推送） |
